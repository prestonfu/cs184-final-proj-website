Progress Summary

Given a text input, we will generate a particle simulator where randomly positioned particles will transform into a 3D cloud point structure representing the input text through swarm intelligence. For example, if the text input was “apple”, the original particles will map to a new position to spell out “apple” in a 3D format. To create our particle simulator, we plan to implement generative point clouds, a flocking algorithm, and ray marching.

So far, we have been able to preprocess images and convert them to a 3D point cloud, create a basic flocking algorithm, and program a ray marching algorithm that is rendered using homework 3’s camera and GUI.

Preliminary Results

  • Generative Point Cloud

    • We have done some image preprocessing. Images, which we collect either manually from the web or through Stable Diffusion 1 via text prompt, have their backgrounds removed and their subjects centered. Then we feed them through OpenAI’s Point-E model, which converts them to a 3D point cloud with RGB values. The result is a numpy (npz) file, which we read as input in Unity.
    • This pipeline has been mostly automated with the limiting factor being cost for a decent LLM API; we prompt an LLM for the original text prompt, and the rest is fed through the other models. Due to the cost of allowing anonymous public users to submit text prompts, we’ve opted to collect some pre-built prompts, and enable a wide variety of selections. We will also consider the option of having other users submit point clouds to a separate form, which we will run on our end and add to the site, to limit the number of submissions. While datasets for point clouds and meshes exist on the Internet, they are fairly small-scale, with only a few dozen generic text prompts.

  • Flocking

    • We wrote a basic flocking algorithm in Unity and C# with Separation, Cohesion, Alignment, Containment, and Arrival forces using the modified Euler method for simulation. For each force we have a different radius and we are computing neighborhoods that ignore boids behind the current boid to simulate line of sight. We’ve also spent time trying to tweak the parameters for each of these to achieve a presentable simulation.
    • We have a script to convert npz files into jsons that can be read by C# or C++ to load in target positions for the Arrival force of each boid.

  • Ray Marching

    • We have programmed a ray marching algorithm, in the hopes of optimizing rendering
    • Also decided to use the camera and GUI codebase from homework 3 to test our implementation.
    • Instead of rendering one image at a time with the camera in one position, we wanted our display to continuously render new images in real time with the ability to also change the camera position in between rendering.
      • We were able to achieve this by editing the rendering files in application.cpp. The resulting display renders images a bit slowly, but we believe we can improve the rendering time by adding BVH to our ray marching algorithm.

Progress Reflection

Our progress is on schedule relative to our original plan which was completing rough implementations for all of our three technical tasks: generative point clouds, flocking, and ray marching. Next, we plan to improve and optimize our implementations. For ray marching, we will implement BVH into our ray marching algorithm to improve rendering times.

An upcoming challenge we have is figuring out how to merge our three codebases together. More specifically, we will have to combine different languages such as C++, OpenGL, and GLCL (the shading language for OpenGL).
